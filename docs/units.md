# Единицы измерения (рост / вес)

Сквозное правило обращения с физическими величинами (рост, вес) — как храним, как показываем, как конвертируем. Касается онбординга ([auth-flow.md](./auth-flow.md)), профиля и настроек приложения.

## Принцип

**В БД — каноничные единицы СИ (метрическая система). Конверсия — только на фронте, для показа и ввода.**

Бэкенд оперирует исключительно единицами СИ и про футы/фунты ничего не знает.

| Величина | Каноничная единица (БД), СИ                                                                |
| -------- | ------------------------------------------------------------------------------------------ |
| Рост     | **метр (м)** — на практике допустимо хранить целыми сантиметрами (см. «точность хранения») |
| Вес      | **килограмм (кг)**                                                                         |

## Конверсия (двусторонняя, на клиенте)

- **Показ:** канон → локальные единицы пользователя, с **округлением** (округление только для отображения).
- **Ввод/сохранение:** значение от пользователя → обратно в канон перед отправкой на бэк.

Поддерживаемые системы отображения:

| Величина | Метрическая | Имперская            |
| -------- | ----------- | -------------------- |
| Рост     | см          | футы-дюймы (ft′ in″) |
| Вес      | кг          | фунты (lb)           |

## ⚠️ Round-trip drift

Нельзя слепо гонять `канон → имперские (округлили) → канон` на каждом сохранении — значение поплывёт:

```
180 см → 5′11″ (округление показа) → 180.3 см → ...
```

**Правило:** конвертировать обратно в канон **только если пользователь реально изменил поле**. Если не трогал — отправляем исходное каноничное значение без изменений. Округление участвует только в отображении, никогда в хранении.

## Настройка единиц

- По умолчанию — из **локали устройства**.
- Можно переопределить в настройках приложения.
- **Решено:** выбор единиц (и прочие настройки) хранится **локально на устройстве** (SQLite), на бэк не синкается — между устройствами не переносится. Передумаем — добавим на профиль; пока MVP-минимум.

## Решено

- **Тип/точность хранения** — рост и вес хранятся как `float`: рост в **сантиметрах**, вес в **килограммах** (бэк, поля `height`/`weight` в `about`). `float` (а не `int`), т.к. ввод в имперских единицах с обратной конверсией даёт дробные канон-значения.
- **Диапазоны валидации** — рост **100–250 см**, вес **30–300 кг**; оба опциональны.
