# Геолокация

Как приложение работает с местоположением: обновление координат, приватность,
формат дистанции в выдаче. Соглашения (пути, токены, конверт ошибок, типы) — см.
[api.md](./api.md). Discovery-лента и её фильтры — отдельный документ; здесь —
только гео-механика, на которую лента опирается.

## Принципы

- **Координаты наружу не отдаём — никогда.** Иначе любой с токеном (Postman и т.п.)
  вытащит адрес человека. Клиент **только присылает** координаты; расстояние между
  людьми **считает бэкенд** и отдаёт уже готовую дистанцию.
- **Гео — отдельная сущность, не поле анкеты.** Локация живёт сама по себе и
  постоянно обновляется, поэтому в `MeetingAd` координат нет (см.
  [meeting-list.md](./meeting-list.md)).

## Обновление локации

### `POST /my-geo`

Записывает текущую локацию пользователя (upsert, одна строка на юзера). Bearer.

```
→ { "lat": 55.751244, "lng": 37.618423 }
← 204
```

- Клиент вызывает **при входе в приложение** и далее по интервалу **раз в 10 минут**.
- Координаты **не возвращаются** ни в этом ответе, ни в каком другом эндпоинте.
- Валидация: `lat` ∈ [-90, 90], `lng` ∈ [-180, 180].

Ошибки: `422 VALIDATION_ERROR`.

## Дистанция и видимость

- Лента использует **сохранённую** локацию зрителя (из последнего `/my-geo`) —
  координаты в запрос ленты **не передаются**.
- Анкеты показываются по **последней координате автора**. Автор вышел из радиуса —
  анкета **скрывается** (без отдельного порога «свежести»: берём последнюю известную точку).
- **Радиус** — параметр ленты, **float, километры** (клиент может конвертировать из
  миль, тогда придёт напр. `1.6`).
- **Дистанция в ответе — целые километры:**
    - `0` — ближе 1 км (фронт показывает «<1 км»);
    - иначе — математическое округление (`1.5 → 2`).
- **Нет гео у зрителя** (запретил доступ / ещё не слал `/my-geo`) → **пустая лента**.
  Симметрично: у кого нет гео — того **не показывают** другим (нет точки → вне выдачи).

## Хранение и поиск (бэкенд)

- `user_geo`: `lat`/`lng` (`double precision`) + `updated_at`, одна строка на
  пользователя (upsert).
- Поиск по расстоянию: дешёвый **bounding-box** префильтр (индекс по `lat`/`lng`) →
  точный **haversine** → фильтр по радиусу → сортировка по дистанции.
- **Без PostGIS** (лишнее расширение для MVP). При росте нагрузки — встроенный
  `cube`/`earthdistance` с GiST-индексом.

## Приватность (резюме)

- Сырые координаты не покидают бэкенд.
- Наружу идёт только **округлённая целая дистанция** (и `0` для «<1 км»).
- Точное место встречи участники согласуют **в чате**, не через API.
