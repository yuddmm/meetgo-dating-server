# Анкета встречи (`meeting-list`)

Контракт **создания и жизненного цикла анкеты** (`MeetingAd`) — намерения
встретиться. Продуктовая логика (роли, отклики, чаты) — в [matching.md](./matching.md);
соглашения (пути, токены, конверт ошибок, типы, НСИ) — в [api.md](./api.md).

Отклики/кандидаты (`MeetingCandidate`) и чаты — отдельные документы. Геолокация и
дистанция — в [geolocation.md](./geolocation.md); **в анкете координат нет**.

## Сущность `MeetingAd`

```json
{
  "id": "<uuid>",
  "authorProfileId": "<uuid>",
  "description": "Иду на кофе в центр, ищу компанию",
  "tags": [{ "id": "<uuid>", "value": "COFFEE" }],
  "status": "ACTIVE",
  "createdAt": "2026-06-22T10:30:00Z",
  "expiresAt": "2026-06-23T10:30:00Z"
}
```

- `authorProfileId` — профиль-автор (роль «автор» = владение активной анкетой, см. matching.md).
- `tags` — теги встречи, НСИ `{ id, value }` из [meeting-tags.md](./meeting-tags.md), ссылка по `id`.
- `status` ∈ `ACTIVE` | `EXPIRED` | `CLOSED`.
- Места/времени встречи в анкете **нет** — участники договариваются в чате; поиск идёт по геолокации.

## Эндпоинты (`/api/v1`, Bearer)

| Метод | Путь | Назначение |
|-------|------|------------|
| GET | `/meeting-tags` | справочник тегов `[{id,value}]` (как `/interests`) |
| POST | `/meeting-list` | создать активную анкету → `201 MeetingAd` |
| GET | `/meeting-list/me` | моя активная анкета (или `404 NO_ACTIVE_AD`) |
| DELETE | `/meeting-list/me` | снять свою активную анкету (→ `CLOSED`) |

### `GET /meeting-tags`

```
← 200 [ { "id": "<uuid>", "value": "COFFEE" } ]
```

### `POST /meeting-list`

```
→ {
    "description": "...",          // 30–500 символов, обязательно
    "tagIds": ["<uuid>"]           // 1–3, существуют в справочнике, без дублей
  }
← 201 <MeetingAd>
← 409 ACTIVE_AD_EXISTS             // уже есть активная анкета (одна на юзера)
← 422 VALIDATION_ERROR             // details: description / tagIds
```

Валидация: `description` обязательно, **30–500** символов; `tagIds` — **1–3**,
все существуют в справочнике тегов, без повторов.

### `GET /meeting-list/me`

```
← 200 <MeetingAd> | 404 NO_ACTIVE_AD
```

Нужно фронту: таб «Лайки» имеет смысл только при активной анкете (matching.md).

### `DELETE /meeting-list/me`

Снять активную анкету вручную (ошибся / пропал интерес) — переводит в `CLOSED`.

```
← 204 | 404 NO_ACTIVE_AD
```

## Жизненный цикл

- **Одна активная анкета на юзера.** `POST` при наличии `ACTIVE` → `409 ACTIVE_AD_EXISTS`.
- **`ACTIVE`** — пока не истекли сутки **и** не возник постоянный чат.
- **`EXPIRED`** — прошло **24 часа** с `createdAt` → анкета пропадает.
- **`CLOSED`** — либо двое согласовали **постоянный чат** (цикл закрыт — триггер в
  чат-модуле), либо анкета снята вручную (`DELETE`).
- После `EXPIRED`/`CLOSED` создавать новую можно (старая больше не `ACTIVE`).

### Пересоздание («Опубликовать снова»)

Клиент **хранит последнюю анкету локально** (SQLite) и при «опубликовать снова»
префилит поля и шлёт обычный `POST /meeting-list` — это **новая анкета с новым
`createdAt`** (время намерения не подделываем). Серверный шаблон не нужен.

## Ошибки

- `409 ACTIVE_AD_EXISTS` — уже есть активная анкета.
- `404 NO_ACTIVE_AD` — нет активной анкеты (на `GET`/`DELETE /meeting-list/me`).
- `422 VALIDATION_ERROR` — `details`: `description` / `tagIds` (в т.ч. неизвестный тег).
