# Справочник интересов (seed)

Стартовый набор интересов для MVP. Это **НСИ** единого типа `{ id, value }` (см. [api.md](./api.md) → «Соглашения»): `value` — стабильный ключ в `SCREAMING_SNAKE_CASE`, перевод — на клиенте по i18n. Бэкенд **сидит** этот список в миграции и отдаёт через `GET /interests`; перевод хранится на фронте, не в БД.

- `key` — стабильный ключ (`value` в НСИ), **не меняется** после релиза.
- `ru` — русский перевод для клиентского i18n (передаётся в репозиторий фронта).
- В онбординге пользователь выбирает **от 1 до 5** интересов ([auth-flow.md](./auth-flow.md)).

> ~50 ключей для старта. Список можно расширять через сид/админку; **ключи не переименовываем** — только добавляем новые (на них могут ссылаться профили).
